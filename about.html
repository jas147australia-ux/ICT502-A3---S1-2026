About this web application

This static local-browser prototype was prepared for ICT502 Scenario 5. It demonstrates core front-end development skills using HTML, CSS and JavaScript while keeping the application easy to run from a normal folder.

Implemented features

  • Income and expense input with categories.
  • Editable transaction table with filtering and search.
  • Monthly and category budget setting.
  • Overspending alerts and status progress bars.
  • Charts generated with JavaScript canvas.
  • CSV, JSON and printable report export.

Technology choices

  • HTML5 for semantic page structure and forms.
  • CSS3 for responsive layout and visual styling.
  • Vanilla JavaScript for business logic and validation.
  • LocalStorage for browser-based data persistence.
  • Canvas for charts without external libraries.

Settings

Changing currency affects display formatting only.

Testing summary

The application was checked for local navigation, form validation, transaction creation, edit and delete functions, monthly filtering, budget alert calculation, chart rendering, and export downloads. The final result showed that the required functions worked in a local browser using saved browser data.
